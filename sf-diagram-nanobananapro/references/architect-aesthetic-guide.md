   1 # Official Salesforce Architect ERD Aesthetic Guide
   2 
   3 Reference for generating ERDs matching the **architect.salesforce.com** visual style.
   4 This is the default style for all ERD diagrams generated by sf-diagram-nanobananapro.

   8 ## Cloud-Specific Color Palettes
   9 
  10 Each Salesforce cloud has its own brand color. The skill auto-detects which cloud to use based on the objects requested.
  11 
  12 | Cloud | Border Color | Fill (~25% opacity) | Detect By Objects |
  13 |-------|--------------|---------------------|-------------------|
  14 | **Sales Cloud** | `#0B827C` (Teal) | Light Teal | Lead, Opportunity, Quote, Campaign, Order |
  15 | **Service Cloud** | `#9E2A7D` (Magenta) | Light Pink | Case, Knowledge, Entitlement, Swarm, Incident |
  16 | **Platform** | `#5A67D8` (Purple) | Light Purple | Account, Contact only (no cloud-specific objects) |
  17 | **Industries/Scheduler** | `#BA4383` (Pink) | Light Pink | ServiceTerritory, WorkType, Shift, ServiceResource |
  18 | **Nonprofit Cloud** | `#032D60` (Navy) | Light Gray | Survey, Donation, Program objects |

  22 ## The Signature Box Style: "Dark Border + Light Fill"
  23 
  24 This is the defining visual pattern of architect.salesforce.com diagrams:
  25 
  26 | Property | Value |
  27 |----------|-------|
  28 | **Border** | Solid 2px in cloud's brand color |
  29 | **Fill** | Same hue at ~25% opacity (translucent) |
  30 | **Corners** | Rounded (8-12px radius) |
  31 | **Text** | Dark gray/black on light fill |
  32 
  33 ### Visual Example
  34 
  35 ```
  36 ┌─────────────────────────┐
  37 │ ░░░░░░░░░░░░░░░░░░░░░░░ │  ← Light translucent fill
  38 │ ░ Opportunity          ░ │  ← Bold logical name
  39 │ ░ Opportunity          ░ │  ← Smaller italic API name
  40 │ ░ • Name               ░ │  ← Key fields with bullets
  41 │ ░ • Amount             ░ │
  42 │ ░ • Close Date         ░ │
  43 └─────────────────────────┘
  44   ↑ Dark solid border
  45 ```

  49 ## Page Structure
  50 
  51 ### 1. Page Title (Top)
  52 
  53 Large bold text centered at the very top:
  54 - Format: `"[Cloud Name] Overview Data Model"`
  55 - Example: `"Sales Cloud Overview Data Model"`
  56 
  57 ### 2. Header Banner
  58 
  59 Below the title, a horizontal banner containing:
  60 
  61 ```
  62 ┌──────────────────────────────────────────────────────────────────────────────┐
  63 │ [salesforce]  SALES CLOUD   │  LEGEND:                                       │
  64 │               Overview      │  ENTITIES        │  RELATIONSHIPS              │
  65 │                             │  [boxes...]      │  [symbols...]               │
  66 └──────────────────────────────────────────────────────────────────────────────┘
  67 ```
  68 
  69 **Left Section:**
  70 - Salesforce cloud logo
  71 - "[CLOUD NAME] Overview" in dark background
  72 
  73 **Center Section - ENTITIES:**
  74 - Cloud Entity (dark border + light fill)
  75 - Related Entity (dark border + light fill)
  76 - External Entity (black border + white fill)
  77 
  78 **Right Section - RELATIONSHIPS:**
  79 - Parent to Child (Many-to-One)
  80 - Required to Required (One-to-One)
  81 - Optional to Optional (Zero-or-One to Zero-or-One)
  82 - Many-to-Many
  83 
  84 ### 3. Main Diagram Area
  85 
  86 The ERD with objects and relationship lines.
  87 
  88 ### 4. Footer
  89 
  90 ```
  91 ┌──────────────────────────────────────────────────────────────────────────────┐
  92 │ ©2024 Salesforce - Subject to change without notice    Last modified: [DATE]│
  93 └──────────────────────────────────────────────────────────────────────────────┘
  94 ```

  98 ## Entity Box Format
  99 
 100 Each object box contains:
 101 
 102 ```
 103 ┌─────────────────────────┐
 104 │ Opportunity             │  ← Bold logical name (larger font)
 105 │ Opportunity             │  ← API name (smaller, italic, gray)
 106 │ • Name                  │  ← Key field 1 with bullet
 107 │ • Amount                │  ← Key field 2 with bullet
 108 │ • Close Date            │  ← Key field 3 with bullet
 109 │ • StageName             │  ← Key field 4 (optional)
 110 └─────────────────────────┘
 111 ```
 112 
 113 ### Recommended Fields by Object
 114 
 115 | Object | Key Fields |
 116 |--------|------------|
 117 | Account | Account Number, Name, Industry, Type |
 118 | Contact | Name, Email, Title, Phone |
 119 | Lead | Name, Company, Status, Rating |
 120 | Opportunity | Name, Amount, Close Date, StageName |
 121 | Campaign | Name, Type, Status, StartDate |
 122 | Case | CaseNumber, Type, Priority, Status |
 123 | Quote | Name, ExpirationDate, Status |
 124 | Order | OrderNumber, EffectiveDate, Status |
 125 | Product2 | Name, ProductCode, IsActive |

 129 ## Relationship Line Labels
 130 
 131 **Always include descriptive text labels on relationship lines.**
 132 
 133 | Relationship Type | Label Examples |
 134 |-------------------|----------------|
 135 | Parent-Child | "child of", "parent of" |
 136 | Lookup | "related to" |
 137 | Junction | "member of", "participant in" |
 138 | Conversion | "converts to", "promoted to" |
 139 | Association | "assigned to", "owned by" |
 140 
 141 ### Line Styling
 142 
 143 | Property | Value |
 144 |----------|-------|
 145 | Stroke width | 1-2px |
 146 | Color | Black (`#000000`) |
 147 | Direction | Straight horizontal or vertical only (NO diagonals) |
 148 | Crow's foot | On "many" side of relationship |
 149 | Bar | On "one" side (required) |
 150 | Circle + Bar | On "one" side (optional) |
